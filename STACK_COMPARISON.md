# Multi-Domain CMS: Stack Comparison

This document evaluates the implementation of a multi-domain CMS across four different technology stacks: **PHP**, **React**, **Vue**, and **Python (FastAPI)**.

## 1. Stack Implementations

### PHP (Reference)
- **Repo**: `/repo-php/`
- **Model**: Server-side directory-based resolution.
- **Key Logic**: `Router::dispatch()` checks `$_SERVER['HTTP_HOST']` and points to a folder in `content/`.
- **Media**: Handles on-the-fly zip extraction and dynamic SVG placeholders via PHP logic.
- **DX**: Standard PHP - immediate feedback on refresh. No build step required for core logic.

### React
- **Repo**: `/repo-react/`
- **Model**: Client-side data-driven resolution.
- **Key Logic**: `useEffect` on mount detects `window.location.hostname`.
- **Components**: Functional components with Tailwind CSS.
- **DX**: High-speed HMR with Vite. Strong type safety with TypeScript.

### Vue
- **Repo**: `/repo-vue/`
- **Model**: Client-side data-driven resolution.
- **Key Logic**: `onMounted` hook detects hostname and updates a reactive `activeSite` ref.
- **Components**: Single File Components (SFC) for clean separation of logic, template, and style.
- **DX**: Vite provides a very smooth development loop.

### Python (FastAPI)
- **Repo**: `/repo-python/`
- **Model**: Server-side templated resolution.
- **Key Logic**: FastAPI route retrieves `Host` header and renders a Jinja2 template.
- **Templating**: Jinja2 allows for clean HTML with logic injectors.
- **DX**: Very explicit routing and easy-to-read logic.

## 2. DX (Developer Experience) Comparison

| Feature | PHP | React / Vue | Python |

| **Hot Reloading** | Natural (on refresh) | Excellent (HMR) | Very Good (Uvicorn reload) |

## 3. AX (AI Agent Experience) Comparison

AX measures how effectively an AI coding agent can interact with, understand, and modify the codebase.

- **PHP (Moderate AX)**: The mixture of PHP logic inside HTML files can sometimes lead to context fragmentation for an AI. However, the directory-based structure is very clear to navigate.
- **React/Vue (High AX)**: Component isolation is the best pattern for AI. An agent can be given a single component to fix without needing the entire app context. TypeScript provides "guardrails" that help the AI catch errors before they even run.
- **Python (Excellent AX)**: Python's readability is synonymous with AI logic. FastAPI's auto-generated docs and clear type annotations make it very easy for an AI to identify entry points and data structures.

## 4. Implementation Recommendations

- **Use PHP** if you have a massive amount of legacy static content or need a robust "plug-and-play" system like WordPress.
- **Use React/Vue** for highly interactive multi-site dashboards or if the "sites" are actually complex mini-webapps.
- **Use Python** if the CMS needs to be integrated with AI processing (e.g., auto-generating text, image analysis) or if you prefer a modern, clean backend API.
