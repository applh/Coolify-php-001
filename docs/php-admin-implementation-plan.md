# PHP CMS Admin API Implementation Plan

This document outlines the proposed architecture for implementing a secure, protected admin area with RESTful API capabilities directly within the PHP runtime. This is intended to supplement or replace the Express.js management layer for environments that require a pure PHP deployment.

## 1. Authentication Strategy (JWT & Sessions)

To provide stateless API access for a single-page application (like Vue 3) as well as traditional server-side rendering, we will implement a hybrid authentication mechanism.

- **Class `Auth` (`/repo-php/class/Auth.php`)**:
  - `login($username, $password)`: Validates credentials.
  - `generateToken($userId)`: Creates a JWT (JSON Web Token) for API clients.
  - `validateToken($token)`: Verifies API requests.
  - `initSession()`: Native PHP session management (`$_SESSION`) for direct browser access.

## 2. API Routing Layer

The main PHP front-controller (`public/index.php`) currently routes all traffic to site templates. We need to intercept requests destined for the admin interface or API endpoints before normal site resolution occurs.

```php
// In public/index.php (Proposed changes)

// Intercept API / Admin traffic
$requestUri = $_SERVER['REQUEST_URI'];

if (strpos($requestUri, '/admin/api/') === 0) {
    // Dispatch to ApiController
    $api = new ApiController();
    $api->handleRequest();
    exit;
} elseif (strpos($requestUri, '/admin') === 0) {
    // Dispatch to AdminController (returns HTML views for admin panel)
    $admin = new AdminController();
    $admin->render();
    exit;
}

// ... existing Router->dispatch() ...
```

## 3. The Controller Classes

### `ApiController` (`/repo-php/class/ApiController.php`)
This class will handle JSON payloads, route to specific actions, and strictly enforce the `Auth::validateToken()` check.

- `handleRequest()`: Parses the HTTP method (`GET`, `POST`, `PUT`, `DELETE`) and path.
- `requireAuth()`: Middleware-like method to abort with `401 Unauthorized` if the token is missing/invalid.
- **Endpoints**:
  - `GET /admin/api/sites`
  - `GET /admin/api/files?site=site1.com&path=/`
  - `POST /admin/api/files` (Save file contents)
  - `DELETE /admin/api/files`

### `AdminController` (`/repo-php/class/AdminController.php`)
Responsible for validating traditional PHP sessions and serving the initial HTML payload for the Vue 3 application. It ensures that unauthenticated users are redirected to a login screen.

## 4. Protected Storage (`/repo-php/data/`)

User credentials and general configuration cannot be stored in the public web root or tenant directories. 

- Create a secure storage directory: `/repo-php/data/` (ensure this is hidden from Nginx/Apache routing).
- Implement a simple Flat-file JSON database or SQLite database exclusively for admin credentials:
  - `data/users.sqlite` or `data/admin.json`.
- The `Auth` class will query this protected storage during the `login()` method.

## 5. Security & CSRF Protection

- All native HTML forms (if any are used outside of the Vue app) will require CSRF tokens generated by the `Auth` class.
- The API will rely on the `Authorization: Bearer <token>` header, heavily mitigating standard CSRF vectors when paired with strict CORS policies.
- Directory traversal checks MUST be enforced when the API reads or writes files (`realpath()` verification against the allowed `content/` path).
