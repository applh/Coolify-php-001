# Node CMS Media Implementation Plan

Because AI Studio cannot directly generate media files (like optimized images, audio, or video) via prompting, the CMS management layer must provide a robust mechanism for users to upload and manage media assets.

This document outlines the architecture and implementation steps required to add a Media Uploader to the Node.js (Express) backend and Vue 3 frontend.

## 1. Backend: Express & Multer integration

Handling file uploads in Node.js requires parsing `multipart/form-data`. The standard library for this in Express is `multer`.

### Implementation Steps
1. **Install Dependencies:**
   ```bash
   npm install multer
   npm install -D @types/multer
   ```

2. **Configure Storage API (`server.ts` or new API route):**
   - Create a Multer disk storage engine to save files directly to the intended PHP destination (e.g., `/repo-php/public/img`, `/repo-php/my-data/[site]/media`, or `/repo-php/content/[site]/assets`).
   - Implement dynamic pathing based on the request (e.g., `req.body.targetPath`) so the Vue front-end can dictate where the image goes, with strict validation to prevent path traversal outside `/repo-php`.

   ```typescript
   import multer from 'multer';
   import path from 'path';

   const storage = multer.diskStorage({
     destination: function (req, file, cb) {
       // Extract and validate target path
       const targetPath = req.body.path || 'public/uploads';
       const fullPath = path.join(repoPhpPath, targetPath);
       
       // Security: Verify path is within repoPhpPath
       if (!fullPath.startsWith(repoPhpPath)) {
         return cb(new Error('Invalid destination path'), '');
       }
       cb(null, fullPath);
     },
     filename: function (req, file, cb) {
       // Sanitize filename and append unique identifier if needed
       const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);
       cb(null, uniqueSuffix + '-' + file.originalname.replace(/[^a-zA-Z0-9.\-_]/g, ''));
     }
   });

   const upload = multer({ 
     storage: storage,
     limits: { fileSize: 10 * 1024 * 1024 }, // 10MB limit
     fileFilter: (req, file, cb) => {
       // Reject dangerous files (.php, .exe, .sh)
       if (file.mimetype.startsWith('image/') || file.mimetype.startsWith('video/')) {
         cb(null, true);
       } else {
         cb(new Error('Only media files are allowed'));
       }
     }
   });

   app.post('/api/media/upload', upload.single('mediaFile'), (req, res) => {
     if (!req.file) return res.status(400).json({ error: 'No file uploaded' });
     res.json({ 
       message: 'Upload successful', 
       filename: req.file.filename,
       path: path.join(req.body.path, req.file.filename).replace(/\\/g, '/')
     });
   });
   ```

## 2. Frontend: Vue 3 Media Manager UI

The frontend needs a way to upload files within the `SiteExplorer` or `SiteEditor` views.

### Implementation Steps
1. **Upload Component:**
   - Create a new component (e.g., `MediaUploader.vue`) that uses a drag-and-drop zone and a standard file input (hidden, triggered by click).
   - Use `FormData` to package the file and the destination path.

   ```javascript
   const uploadFile = async (file, targetPath) => {
     const formData = new FormData();
     formData.append('mediaFile', file);
     formData.append('path', targetPath);

     const res = await fetch('/api/media/upload', {
       method: 'POST',
       body: formData
     });
     
     if (!res.ok) throw new Error('Upload failed');
     return await res.json();
   };
   ```

2. **Integration into Explorer:**
   - In `SiteExplorer.vue`, add an "Upload File here" button when a directory is currently selected.
   - Upon successful upload, refresh the directory listing via the existing `explore(currentPath)` function.

3. **Gallery View / Clipboard:**
   - In the Explorer view, if an item is an image (`.png`, `.jpg`, `.webp`, `.svg`, etc.), display a thumbnail preview instead of just a text icon.
   - Provide a "Copy Path" button against media files so users can quickly paste it into their HTML/Markdown content (`<img src="/media/uploaded-image.jpg">`).

## 3. Workflow Summary for AI Studio users
Because AI Studio can't write binary media, the user workflow will be:
1. User interacts with AI Agent to build PHP templates and general website layout.
2. User opens the CMS Dashboard (Live Preview).
3. User navigates to the target site folder using the Explorer.
4. User selects "Upload", pulls media from their local machine (or generated by separate AI tools like Midjourney/DALL-E).
5. User copies the uploaded file path and pastes it into their template block using the Site Editor.
